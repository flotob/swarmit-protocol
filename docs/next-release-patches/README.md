# Next Release Patch Specs

This directory collects draft patch specifications intended for the next protocol release after the current v1 line.

These documents are working design inputs:

- they are not yet part of the canonical protocol spec
- they may introduce breaking changes that are not suitable for retrofitting into an existing v1 deployment
- they should be concrete enough to implement later once we decide to adopt them

When a patch is adopted, the relevant normative text should be merged into the canonical spec, contract spec, schemas, and implementation notes as appropriate.
